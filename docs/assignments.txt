Your Components
Job Scraping Agent — /agents/job_scraping_agent.py
Collect job/internship listings from LinkedIn and Indeed using search filters (keywords, location, job type, experience level)
Extract: company name, job title, description, required skills, location, salary, deadline, contact info
Assign unique job_id to each record; standardize fields across platforms
Model: gpt-4o-mini — do NOT upgrade without lead approval
Tools to call: scrape_linkedin_jobs, scrape_indeed_jobs

Job Verification Agent — /agents/job_verification_agent.py
Detect duplicate postings across platforms
Verify company legitimacy using public records (verify_company tool)
Identify expired postings by comparing posted_date vs application_deadline
Flag suspicious listings (spelling errors, unusual payment requests, invalid contact links)
Output verified_status: verified / rejected / flagged_for_review per job

Error Handling (both agents)
API connection failures: retry with exponential backoff
Rate limiting: implement queue and schedule retries
Incomplete job data: log and flag for manual review — never crash the pipeline

Reorganize and highlight CV content to align with job keywords — NEVER invent skills, experience, or achievements (hard constraint)

Tool to call: generate_resume

Cover Letter Agent — /agents/cover_letter_agent.py
Generate personalized cover letter using profile, job description, and company info
Tone: professional, tailored to industry; coordinate with Resume Agent for consistency
Handle missing company info by generating a generic but professional letter
Tool to call: generate_cover_letter

Provide statistics: total applied, response rate, interview rate
Tool to call: update_application_status

Shared Rules for All Team Members
Rule
Detail
Branch naming
Always use the exact branch name assigned to you — no custom names
Commit messages
Use conventional commits: feat: / fix: / chore: / test: / docs: followed by description
PR deadline
All PRs must be open on GitHub at least 2 days before your phase end date to allow review time
No hardcoded secrets
Use environment variables for all API keys and OAuth tokens — violations will block your PR merge
Tests first
Write at least 1 unit test per function before opening your PR — no test-free PRs accepted
Blockers
If you are blocked for more than 4 hours, message the Lead (Member 1) immediately — do not wait
Model discipline
Do not change the assigned model (gpt-4o-mini vs gpt-4o) without Lead approval — it affects cost KPIs
Interface spec
Check /docs/tool_interface_spec.md (written by Lead, available Week 1 Day 3) before writing any tool
Factual accuracy
Resume/Cover Letter content must never include invented skills, experience, or credentials — this is a hard constraint across the system
Data privacy
All PII (CVs, contact info, tokens) must be encrypted at rest (AES-256) and in transit (TLS 1.3) per SRS 5.2
